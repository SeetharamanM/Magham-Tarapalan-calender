# கூகுள் காலெண்டர் அமைப்பு வழிகாட்டி

## முன்னுரை

இந்த வழிகாட்டி உங்கள் மக நட்சத்திர தரபாலன் காலெண்டர் பயன்பாட்டை கூகுள் காலெண்டருடன் இணைக்க உதவுகிறது.

## கட்டம் 1: கூகுள் கிளவுட் ப்ராஜெக்ட் உருவாக்குதல்

1. **கூகுள் கிளவுட் கன்சோல் திறக்கவும்:**

   - <https://console.cloud.google.com/> செல்லவும்
   - உங்கள் கூகுள் கணக்குடன் உள்நுழையவும்

2. **புதிய ப்ராஜெக்ட் உருவாக்குதல்:**

   - மேலே உள்ள ப்ராஜெக்ட் தேர்வுபுலத்தை கிளிக் செய்யவும்
   - "NEW PROJECT" கிளிக் செய்யவும்
   - ப்ராஜெக்ட் பெயர்: "Magha Tarapalan Calendar"
   - "CREATE" கிளிக் செய்யவும்

## கட்டம் 2: Google Calendar API இயக்குதல்

1. **API நூலகத்திற்குச் செல்லவும்:**

   - இடது பக்கமெனுவில் "APIs & Services" → "Library" கிளிக் செய்யவும்

2. **Google Calendar API தேடுதல்:**

   - "Google Calendar API" தேடவும்
   - முடிவுகளில் இருந்து Google Calendar API தேர்ந்தெடுக்கவும்
   - "ENABLE" கிளிக் செய்யவும்

## கட்டம் 3: OAuth 2.0 உரிமைகள் உருவாக்குதல்

1. **OAuth ஒப்புதல் திரையை உருவாக்குதல்:**

   - "APIs & Services" → "OAuth consent screen" கிளிக் செய்யவும்
   - User Type: "External" தேர்ந்தெடுக்கவும்
   - "CREATE" கிளிக் செய்யவும்

2. **பயன்பாட்டு தகவல்களை நிரப்புதல்:**

   - App name: "Magha Tarapalan Calendar"
   - User support email: உங்கள் மின்னஞ்சல்
   - Developer contact information: உங்கள் மின்னஞ்சல்
   - "SAVE AND CONTINUE" கிளிக் செய்யவும்

3. **குறியீடுகள் (Scopes) சேர்த்தல்:**

   - "ADD OR REMOVE SCOPES" கிளிக் செய்யவும்
   - தேடுபுலத்தில் "calendar" தட்டவும்
   - "Google Calendar API" → "https://www.googleapis.com/auth/calendar.events" தேர்ந்தெடுக்கவும்
   - "ADD TO TABLE" → "SAVE AND CONTINUE" கிளிக் செய்யவும்

4. **சோதனை பயனர்களை சேர்த்தல்:**

   - "ADD USERS" கிளிக் செய்யவும்
   - உங்கள் கூகுள் மின்னஞ்சலை சேர்க்கவும்
   - "SAVE AND CONTINUE" கிளிக் செய்யவும்
   - "BACK TO DASHBOARD" கிளிக் செய்யவும்

## கட்டம் 4: உரிமைகள் உருவாக்குதல்

1. **உரிமைகள் பக்கத்திற்குச் செல்லவும்:**

   - "APIs & Services" → "Credentials" கிளிக் செய்யவும்

2. **API Key உருவாக்குதல்:**

   - "+ CREATE CREDENTIALS" → "API key" கிளிக் செய்யவும்
   - API key நகலெடுக்கவும்
   - "RESTRICT KEY" கிளிக் செய்யவும்
   - API restrictions: "Google Calendar API" மட்டும் அனுமதிக்கவும்
   - "SAVE" கிளிக் செய்யவும்

3. **OAuth 2.0 Client ID உருவாக்குதல்:**

   - "+ CREATE CREDENTIALS" → "OAuth client ID" கிளிக் செய்யவும்
   - Application type: "Web application"
   - Name: "Magha Tarapalan Web App"
   - Authorized JavaScript origins: `http://localhost:3000`
   - Authorized redirect URIs: `http://localhost:3000`
   - "CREATE" கிளிக் செய்யவும்
   - Client ID நகலெடுக்கவும்

## கட்டம் 5: பயன்பாட்டில் உரிமைகளை உள்ளிடுதல்

1. **google-calendar-config.js கோப்பைத் திறக்கவும்**

2. **உங்கள் உரிமைகளை பதிலீடு செய்யவும்:**

   ```javascript
   const GOOGLE_CALENDAR_CONFIG = {
       CLIENT_ID: 'உங்கள் கிளையன்ட் ID இங்கே வைக்கவும்',
       API_KEY: 'உங்கள் API Key இங்கே வைக்கவும்',
       DISCOVERY_DOCS: ['https://www.googleapis.com/discovery/v1/apis/calendar/v3/rest'],
       SCOPES: 'https://www.googleapis.com/auth/calendar.events'
   };
   ```

## கட்டம் 6: பயன்பாட்டைப் பயன்படுத்துதல்

1. **பயன்பாட்டை மீண்டும் தொடங்குதல்:**

   ```bash
   npm run dev
   ```

2. **கூகுள் உள்நுழைவு:**

   - "கூகுள் உள்நுழைவு" பொத்தானை கிளிக் செய்யவும்
   - உங்கள் கூகுள் கணக்கில் உள்நுழையவும்
   - அனுமதி கேட்கும்போது "Allow" கிளிக் செய்யவும்

3. **காலெண்டர் ஒத்திசைவு:**

   - "இந்த மாதத்தை ஒத்திசை" பொத்தானை கிளிக் செய்யவும்
   - நிகழ்வுகள் உங்கள் கூகுள் காலெண்டரில் உருவாக்கப்படும்

## மாற்று வழி: ICS ஏற்றுமதி

Google உள்நுழைவு இல்லாமல், நீங்கள் ICS கோப்பாக ஏற்றுமதி செய்யலாம்:

1. "ICS கோப்பாக ஏற்றுமதி" பொத்தானை கிளிக் செய்யவும்
2. கோப்பு பதிவிறக்கம் செய்யப்படும்
3. Google Calendar → இடது பக்கம் → "+" பொத்தான் → "Import" கிளிக் செய்யவும்
4. பதிவிறக்கம் செய்த ICS கோப்பைத் தேர்ந்தெடுக்கவும்

## பாதுகாப்பு குறிப்புகள்

- உங்கள் API Key மற்றும் Client ID ஐ ஒருபோதும் பொதுவாகப் பகிரவும்
- பயன்பாட்டை தயாரிப்புக்கு மட்டுமே பயன்படுத்தினால், உரிமைகளை கட்டுப்படுத்தவும்
- வழக்கமான இடைவெளிகளில் உங்கள் ப்ராஜெக்ட் செயல்பாட்டைச் சரிபார்க்கவும்

## சிக்கல் தீர்வு

### பிழை: "API key not authorized"

- Google Calendar API இயக்கப்பட்டுள்ளதா சரிபார்க்கவும்
- API key கட்டுப்பாடுகள் சரியாக அமைக்கப்பட்டுள்ளதா சரிபார்க்கவும்

### பிழை: "OAuth consent screen required"

- OAuth ஒப்புதல் திரை முழுமையாக உருவாக்கப்பட்டுள்ளதா சரிபார்க்கவும்
- சோதனை பயனர்கள் பட்டியலில் உங்கள் மின்னஞ்சல் சேர்க்கப்பட்டுள்ளதா சரிபார்க்கவும்

### பிழை: "Redirect URI mismatch"

- Authorized redirect URIs ல் `http://localhost:3000` சேர்க்கப்பட்டுள்ளதா சரிபார்க்கவும்
- உங்கள் பயன்பாட்டின் URL சரியாக உள்ளதா சரிபார்க்கவும்

மேலும் உதவிக்கு, Google Cloud Platform ஆவணங்களைப் பார்க்கவும்.
